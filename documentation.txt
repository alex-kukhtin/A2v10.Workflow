
DeferExec
=========
1. Внутри скрипта есть объект Deferred. Создается как new Deferred().
2. В нем есть метод executeSql(proc:string, prms?:object):void;
3. Все, что будет выполнено этим скрпитом "отоложится" и будет сохранено при сохранении экземпляра.

Сохранять можно только в том-же Source (connectionString), в котором работает Workflow. 
Это потому, что оно будет выполнено в той же транзакции, что и a2wf.[Instance.Update].

prms: плоский объект, без вложенных и массивов. Это параметры для хранимой процедуры. Связывание по именам.

При сохранении к процедуре (кроме параметров из prms) будут добавлены два служебных параметра:
- @InstanceId - Id экземпляра
- @Activity - Id активити из которого вызывался executeSql.
